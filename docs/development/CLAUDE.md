## Project Overview

AgentForge is an AI-powered multi-agent orchestration platform. It replaces the single-AI coding assistant with a structured AI engineering team (Team Lead, Backend, Frontend, QA, Security, DevOps) that collaborates through LangGraph-driven workflows.

## Tech Stack

- **Frontend:** Next.js 15 (App Router), React 19, TypeScript, Tailwind CSS, shadcn/ui
- **Backend:** FastAPI (Python 3.11+), Pydantic v2, asyncpg, redis-py
- **Agent:** LangGraph, OpenAI SDK, Anthropic SDK, Google Generative AI SDK
- **Database:** PostgreSQL 16 + pgvector, Prisma ORM (TypeScript client)
- **Infra:** Docker Compose (local), Turborepo, pnpm 9
- **Auth:** Local JWT authentication (bcrypt password hashing, PyJWT tokens)

## Naming Conventions

### Files
- **TypeScript/React:** `kebab-case.ts`, `kebab-case.tsx`
- **Python:** `snake_case.py`
- **Components:** `PascalCase.tsx` (e.g., `TeamGrid.tsx`, `AgentMessage.tsx`)
- **API routes (Next.js):** `route.ts` in `app/api/v1/resource/`
- **API routes (FastAPI):** `snake_case.py` with `@router` decorators
- **DB migrations:** `YYYYMMDD_HHMMSS_description.sql` (auto-generated by Prisma)

### Code
- **TypeScript:** PascalCase for types/interfaces, camelCase for variables/functions
- **Python:** PascalCase for classes/Pydantic models, snake_case for functions/variables
- **Database:** `snake_case` for tables and columns
- **Environment variables:** `UPPER_SNAKE_CASE`
- **Git branches:** `feat/description`, `fix/description`, `chore/description`

## Forbidden Patterns

- Do NOT hardcode model names — use `model_registry.py` lookup
- Do NOT write system prompts as raw strings in node code — use `prompts/` template files
- Do NOT commit `.env` files, API keys, or secrets
- Do NOT use `print()` for debugging — use `logger` from `core/logging.py`
- Do NOT use synchronous HTTP clients for AI provider calls — use `httpx.AsyncClient`
- Do NOT add new agent roles without updating `AGENT_ROLES.md` and the role enum in the schema
- Do NOT modify prompt templates without bumping the version number in the prompt file header

## Preferred Libraries Per Layer

| Layer | Library | Why |
|-------|---------|-----|
| HTTP client (FE) | `fetch` via `lib/api-client.ts` | No extra dependency, typed wrapper |
| HTTP client (BE) | `httpx` | Async, connection pooling |
| AI provider calls | `openai`, `anthropic`, `google-genai` SDKs | Official, maintained |
| Agent graphs | `langgraph` | Directed graph, state persistence, interrupts |
| DB access (BE) | `asyncpg` directly | Pool management, raw queries for perf |
| DB access (FE types) | `prisma` client | Generated types, migration management |
| Validation (BE) | `pydantic v2` | Schema validation, serialization |
| State management (FE) | React `useReducer` + Context | Simple, no external dep for current scale |
| Testing (FE) | `vitest` + `@testing-library/react` | Fast, modern |
| Testing (BE) | `pytest` + `pytest-asyncio` + `pytest-httpx` | Async support, HTTP mocking |
| E2E testing | `@playwright/test` | Cross-browser, reliable |

## Code Style

- **TypeScript:** strict mode, explicit return types on functions, no `any`
- **Python:** type hints on all function signatures, no `Optional[x]` — use `x | None`
- **Imports:** group stdlib → third-party → local, alphabetical within groups
- **Error handling:** use `Result[T, E]` pattern (BE) or custom exception types
- **Agent nodes:** each node is a standalone async function: `async def node_name(state: AgentState) -> AgentState`
- **API handlers:** FastAPI `async def` with Pydantic models for request/response

## Running Dev Server

```bash
# Terminal 1: Infrastructure
docker compose up -d

# Terminal 2: Dev servers (runs both Next.js and FastAPI)
pnpm dev

# Or individually:
pnpm dev:web    # Next.js on :3000
pnpm dev:api    # FastAPI on :8000

# Watch mode for packages
pnpm dev:packages
```

## Common Task Recipes

### Add a new Agent Role
1. Add role to `AgentRole` enum in `packages/types/index.ts`
2. Add role to `agents` table CHECK constraint in Prisma schema
3. Create `apps/api/agents/prompts/{role}.jinja2` with system prompt template
4. Create `apps/api/agents/nodes/{role}_node.py` with node implementation
5. Register node in `apps/api/agents/graph.py` (add node + add edges)
6. Add role to `AGENT_ROLES.md` and `AGENT_PROMPTS.md`
7. Create a test file: `tests/test_{role}_node.py`

### Add an API Endpoint
1. Add Pydantic models in `apps/api/models/schemas.py`
2. Create or update `apps/api/app/routes/{resource}.py`
3. Register router in `apps/api/app/main.py`
4. Add tests in `tests/test_routes/test_{resource}.py`
5. Update `docs/API.md` with the new endpoint

### Add a DB Table
1. Update `packages/db/schema.prisma` with the new model
2. Run `pnpm db:migrate --name description`
3. Update `docs/SCHEMA.md` with new table definition

### Add a WebSocket Event
1. Add event type to `WSEventType` enum (TS + Python)
2. Add handler in `apps/api/app/ws/task_stream.py`
3. Add client dispatch in `apps/web/lib/ws-client.ts`
4. Document event in `docs/API.md` WebSocket section
