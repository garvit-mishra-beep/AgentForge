# Demo Comparison Layer — Product Requirements Document

**Status:** Draft v1
**Owner:** CPO
**Target Users:** First-time visitors, investors, anyone asking "why not ChatGPT?"
**Value Proposition:** "See exactly what a single AI model misses that AgentForge catches."

---

## 1. Problem Statement

The current demo shows an animated pipeline: 5 message boxes appearing one by one with model names, token counts, and timestamps.

A user watches this and thinks: "Cool, it works. But why is this better than asking ChatGPT the same question?"

**The demo answers *how* but not *why.** *

The product's existential question is "why not ChatGPT?" and the demo doesn't address it. This is the single biggest conversion problem — the demo is the moment users decide whether AgentForge is interesting or not, and it currently fails.

---

## 2. User Story

> As a developer who uses ChatGPT daily,
> I want to see what ChatGPT would miss that AgentForge catches,
> So that I understand why I should add AgentForge to my workflow.

---

## 3. Success Metrics

| Metric | Target | Why |
|--------|--------|-----|

## 5. Demo Script

```
[Self-advancing. ~45 seconds total.]

1. [0s] "See what ChatGPT misses."
2. [3s] "Here's a JWT auth function generated by ChatGPT. Looks fine at first glance."
3. [8s] Code appears. "But there are issues ChatGPT didn't catch."
4. [12s] Issues fade in one by one. "Hardcoded secret. Missing error handling. SQL injection risk."
5. [20s] Comparison box appears. "ChatGPT would have shipped 3 bugs and 0 tests."
6. [25s] "AgentForge caught all 3 bugs and generated 12 test cases."
7. [30s] "This is what happens when 4 specialized agents review your code instead of 1."
8. [35s] CTAs appear. "Try it with your code →"
9. [40s] End.
```

User can click to advance faster. Auto-advance pauses if user appears inactive.

## 6. Frontend Components

### DemoStep
- Generic step container with title, content area, and optional animation
- Steps auto-advance with configurable timing
- User can click anywhere to advance

### DemoCodeBlock
- Monospace code block with optional line highlighting
- Appears with typewriter animation (or instant — typewriter may feel slow)
- Line numbers visible
- Can highlight specific lines (e.g., line 3 for hardcoded secret)

### DemoIssueReveal
- Issues appear one by one with a fade-in + slide effect
- Each shows: severity icon, title, one-line description
- Staggered timing: 0.5s between each

### DemoComparisonPanel
- Side-by-side: "ChatGPT would have shipped" vs "AgentForge caught"
- Left side shows ✅/❌ per item
- Right side shows 🔴/🟢 per item
- Bottom summary text: "Without AgentForge: X bugs + 0 tests. With AgentForge: caught X, generated Y tests."

### DemoCTA
- "Try It With Your Code" — primary, blue, links to Quick Review
- "Watch Again" — secondary, ghost button, resets demo

## 7. Copy Requirements

| Element | Copy | Rationale |
|---------|------|-----------|
| Demo entry title | "See what ChatGPT misses" | Direct. Provocative. Memorable. |
| Demo entry subtitle | "We asked 4 AI agents to review code generated by ChatGPT. Here's what happened." | Sets up the premise immediately |
| Step 1 title | "Here's what ChatGPT generated" | Frames the baseline |
| Tension builder | "Looks good, right? But there are issues ChatGPT didn't catch..." | Creates "aha" moment |
| Results header | "AgentForge found 4 issues in that same code" | Proves the premise |
| Comparison summary | "ChatGPT would have shipped: X bugs + 0 tests. AgentForge caught: X bugs + Y tests." | One sentence that proves the value |
| Technical expand | "How the review worked" — "4 specialized agents doing their job" | Accessible, not technical |

**Never write:**
- "Multi-agent pipeline"
- "Model orchestration"
- "Token count"
- "Agent roles (Lead/Builder/Reviewer/Tester)"
- Any model name (qwen, phi, llama)

## 8. Animation Requirements

| Element | Animation | Timing |
|---------|-----------|--------|
| Entry title | Fade in | 0.3s |
| Code block | Slide in from right | 0.5s |
| Tension text | Fade in | 0.3s delay |
| Issues | Staggered slide-in from left | 0.4s each, 0.8s gap |
| Comparison box | Scale up from center (subtle) | 0.5s |
| "See what" highlight | Subtle glow on "misses" | Constant |
| CTA buttons | Fade in | 0.3s at end |

Animations should feel informative, not flashy. No particle effects, no confetti.

## 9. Acceptance Criteria

- [ ] Demo starts playing immediately (no click required after page load)
- [ ] Demo completes in <45 seconds (auto-advance)
- [ ] User can click to advance faster
- [ ] First thing user sees is "Here's what ChatGPT generated"
- [ ] Issues are revealed one by one with staggered animation
- [ ] Comparison panel shows both what single AI misses and what was caught
- [ ] No model names, token counts, or pipeline jargon visible
- [ ] "Try It With Your Code" links to Quick Review
- [ ] Technical detail section is collapsed by default
- [ ] Works on mobile (stacked layout, shorter timing)
- [ ] Multiple demo scenarios available (JWT, API, React, SQL)
- [ ] Scenario selection changes both the input code and the issues found

---

## 10. What NOT to Build

| Feature | Rejection Reason |
|---------|-----------------|
| Live pipeline execution | Too slow, risks failure, adds complexity |
| Real-time model output | Shows infrastructure instead of value |
| Interactive code editing | Not the purpose of the demo |
| Performance metrics | Premature — users need to care first |
| Side-by-side model comparison | Confuses the message. Focus on ChatGPT only. |
| Skip to any step | Linear flow builds tension. Let user click to advance instead. |
| Audio/narration | 80% of users watch demos muted. Text + animation is sufficient. |

---

## 11. Edge Cases

| Case | Handling |
|------|----------|
| Slow connection | Use pre-loaded static data. No API calls for demo content. |
| User clicks rapidly | Debounce to prevent skipping key frames. |
| User navigates away mid-demo | No persistence needed. Demo restarts on next visit. |
| Multiple demo scenarios | Show a selector before demo starts: "Choose a scenario" |
| Screen reader | All content is semantic HTML with aria-labels. Animations respect prefers-reduced-motion. |
| Auto-advance toggle | Small toggle at bottom: "Auto-play" on/off. Off by default for repeat viewers. |
